**Angle:** Personal narrative. Morning to end of day. Shows the real workflow.
**Delivery:** Calm, direct. Like you're explaining to another founder over coffee.

---

**[0-3s — Hook]**
"Most founders spend 3 hours a day on tasks AI should be running."

**[4-15s — Problem]**
"Research. Outreach. Writing cold emails. Drafting posts. These are all systems problems, not creativity problems. You don't need to do them. You need an OS that does them while you focus on decisions only you can make."

**[16-50s — Body]**
"Here's my actual morning. I open Ultron's dashboard. The research module ran overnight and left 9 leads in the queue — all scored above 8. The sales engine drafted emails for each one. I read them, approve 7, skip 2. Done in 11 minutes.

By afternoon, the content engine has drafted 3 LinkedIn posts based on what the research module found. I review. Edit one. Approve two. They go to the queue.

I didn't write anything. I made judgment calls."

**[51-65s — CTA]**
"Follow if you want to see how to build this kind of system with Claude as the core reasoning engine."

**[66-80s — Payoff]**
"The goal isn't to use less AI. It's to stop using it like a writing assistant and start using it like an OS."

## Reels Script 3 — "Why I Route Different Tasks to Different Claude Models"
**Angle:** Model routing. Technical but accessible. Directly on ICP for AI practitioners.
**Delivery:** Confident, slightly opinionated. This is a take, not a tutorial.

---

**[0-3s — Hook]**
"You're using one AI model for everything. That's the bottleneck."

**[4-15s — Problem]**
"Claude Sonnet is exceptional at multi-step reasoning. GPT-4o is faster on short formatting tasks. Gemini handles 200K token context windows. These aren't the same tool. Using one for everything is like hiring one person to do your sales, research, and engineering."

**[16-45s — Body]**
"In Ultron, every module routes to a different model based on the task.

Research tasks — complex, multi-step, source-heavy — go to Claude. The reasoning quality shows.

Speed tasks — reformatting data, simple classifications, short drafts — go to GPT-4o. Lower cost, faster output.

Long-document processing — ingesting decks, contracts, long threads — routes to Gemini.

The routing logic is built into the dispatch layer. You don't configure it per task. You set the rules once and the OS handles it."

**[46-58s — CTA]**
"Follow if you're building production AI systems and want to get past single-model thinking."

**[59-70s — Payoff]**
"The founders who figure out model routing first will have a real cost and quality advantage. Most haven't started."
