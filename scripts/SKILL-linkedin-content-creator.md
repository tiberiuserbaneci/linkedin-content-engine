# LinkedIn Content Creator Skill v2.0

## OVERVIEW

**Purpose:** Generate viral, save-worthy LinkedIn content for Ultron's target ICP (solo founders, small teams 1-10, US/UK, B2B SaaS/services adopting AI to replace headcount).

**Output:** Always produces TWO things simultaneously:
1. **HTML Artifact** (1080×1350px, self-contained, ready to export as PNG/PDF)
2. **Post Copy** (hook + body + first comment CTA)

**Process:** ANALYZE → SELECT LAYOUT → SCORE (85+ target) → BUILD → WRITE COPY → OUTPUT

---

## PART 1: 360BREW ALGORITHM (LinkedIn Ranking Intelligence)

### Algorithm Overview

LinkedIn's 150B-parameter model evaluates content semantically as an ecosystem:
- Profile alignment
- Engagement patterns  
- Topic authority
- Dwell time
- Save patterns

**Highest signals (in order):**
1. **SAVES** — 4-6x reach multiplier (especially 24-72h late saves)
2. Topic authority (consistent 60-90 days in 2-3 niches)
3. Profile-content alignment
4. Substantive comments (9+ words, personal experience, lexical diversity)
5. Dwell time (people read fully, not scroll past)
6. Specific data, frameworks, actionable steps

### Mandatory Rules

**POST FORMAT:**
- Hook: Exactly 2 lines. Stops the scroll. Contrarian, data point, or specific situation.
- Body: 14+ short paragraphs. Each 10-19 words. Active voice always.
- End: "Repost if [specific founder situation]" OR save prompt
- No hashtags. No links in post body (CTA in first comment only).
- Character target: 1,242–2,500

**BEST POSTING TIME:** 11:00–13:00 GMT, any day (Romania timezone context: 13:00–15:00 RO)

**BANNED WORDS:** leverage, optimize, seamless, delve, robust, ecosystem, revolutionary, game-changing, transformative, holistic, synergy, empower, unlock, journey

**PENALIZES:**
- Engagement pods
- Hashtags (zero effect)
- Volume over quality
- AI slop (em dashes, "moreover", "delve")
- Profile-content mismatch
- Links in post body
- Walls of text

## PART 2: SCORING FRAMEWORK (Before Building)

**Score every concept. Target 85+. Rebuild if below 70.**

### Dimension 1: Virality (Scroll-Stop Potential)

| Signal | What it measures | Weight |
|--------|-----------------|--------|
| ScrollStop | Does line 1 make someone stop? | 25% |
| Clarity3s | Understood in 3 seconds? | 20% |
| Novelty | New angle or just another take? | 20% |
| EmotionalTrigger | Fear/FOMO/relief/pride activated? | 20% |
| FormatFit | Right format for this content type? | 15% |

**Score 0-100 per signal. Weighted average = Virality score.**

### Dimension 2: Save-Worthiness (Algorithm Gold)

| Signal | What it measures | Weight |
|--------|-----------------|--------|
| UtilityDensity | Actionable per cm²? | 25% |
| Reusability | Will they return to this? | 25% |
| Compression | Maximum value, minimum words? | 20% |
| StructureClarity | Navigate without reading everything? | 15% |
| Actionability | Can start using this today? | 15% |

**Score 0-100 per signal. Weighted average = Save-Worthy score.**

### Dimension 3: Visual Efficiency

| Signal | What it measures | Weight |
|--------|-----------------|--------|
| HierarchyStrength | Eye path obvious? | 25% |
| ContrastControl | Dark/light alternation works? | 20% |
| SpacingBalance | Breathing room without waste? | 20% |
| Consistency | Colors, fonts, spacing systematic? | 20% |
| NoiseLevel | Nothing competes for attention? | 15% |

**Score 0-100 per signal. Weighted average = Visual score.**

### Dimension 4: System Thinking (Authority Signal)

## PART 3: DESIGN ARCHITECT (Layout Selection)

### Content Analysis (Mandatory Before Layout)

1. **Content type?** List / process / comparison / framework / data / story / hierarchy
2. **Volume?** 3 items vs 12 items = different layout
3. **Dominant visual idea?** One thing must dominate.
4. **Desired viewer action?** Save / share / comment

### Layout Library (Select Based on Analysis)

| Layout | Best For | Content Type |
|--------|----------|--------------|
| **HERO_GRID** | 5-9 items, icon-based | List with visual emphasis |
| **TIMELINE** | Sequential process | Step-by-step, numbered flow |
| **COMPARISON** | Two or more options | Clear winner column |
| **PYRAMID** | Hierarchy | Rare at top, foundational at base |
| **DASHBOARD** | Data-heavy | Large callout numbers, metrics |
| **ACROSTIC** | Framework acronym | Letters spell a word |
| **MODULAR_CARDS** | 4-6 equal concepts | Grid of equal-weight cards |
| **MOSAIC** | Complex mixed content | Asymmetric grid |
| **FEAR_CARD** | What breaks without this | 5 columns, red top border |
| **MAGAZINE** | Editorial | Large image + text column |

### Three Master Templates

**TEMPLATE 1: LIGHT EDITORIAL** (Claude-branded, comparisons, how-to)
- Background: #F8F6F1 warm cream
- Header: Orange section label + large black title + gray subtitle
- Sections: Labeled with dividers
- Cards: White on warm background
- Footer: Black banner with CTA

**TEMPLATE 2: DARK HERO + LIGHT BODY** (Ultron system, agent explanations, bold statements)
- Top 30-35%: #111 background, massive title (line 1 white, line 2 orange)
- Stat row: 4 metrics in orange/gray
- Remaining 65-70%: #F8F6F1 background with feature grid/table
- Footer: Black banner

**TEMPLATE 3: STRUCTURED MIXED** (comparisons, setup guides, decision frameworks)
- Background: #F8F6F1
- Large title (line 1 black, line 2 orange)
- Comparison table first
- Side-by-side explanation cards
- Feature grid or callout boxes
- Footer: Black banner

### Design Principles (Non-Negotiable)

✓ Every visual is different  
✓ Typography as design: titles 72–120px minimum  
✓ Maximum contrast: dark next to light section  
✓ One dominant element per section  
✓ Skimmable in 3 seconds, saveable for 30 minutes  
✗ No green, ever  
✗ No gradients, no shadows, no rounded cards (max 4px border-radius)  
✗ No placeholder text or lorem ipsum

## PART 4: COPY ARCHITECT (Post & CTA Writing)

### Voice Rules

- Plain everyday language. Grade 6-7 readability (Hemingway target).
- Banned words: leverage, optimize, seamless, delve, robust, ecosystem, revolutionary, game-changing, transformative, holistic, synergy, empower, unlock, journey
- No em dashes (-), en dashes (-), curly quotes, ellipsis character
- Use `->` for lists, not bullets
- Numbers always digits (7 not seven, 3x not three times)
- Name the exact pain. Not "save time" — "stop doing 3-hour Monday reporting"

### Hook Formulas (Proven on ICP)

```
CONTRARIAN:    "[Common belief] is wrong. Here's what actually works:"
DATA LEAD:     "X% of [ICP] do [thing] manually. They lose [specific cost]."
FOLDER TREE:   Show structure visually in text. People stop to read.
NUMBERED LIST: "7 things your [role] should automate this week:"
SPECIFIC SIT:  "It's 9AM Monday. You have 47 unread emails. Here's what Ultron does first."
```

### Post Structure

```
Line 1-2:  HOOK (stop the scroll)
[blank]
Body:      14+ short paragraphs. Each 10-19 words. Active voice.
           Each paragraph = one idea.
[blank]
End:       "Repost if [specific founder situation]" OR "Save this. You'll need it."
[no CTA in post body]
```

### First Comment CTA (Put Here, Not in Post)

```
FORMAT:
"Ultron [does exactly what the post described].
-> [relevant URL]
[1-2 sentences of specific value, no selling language]"

ROUTING:
Automations/agents       -> 51ultron.com/blueprint/
ROI/cost/savings         -> 51ultron.com/calculator/
Competitor/intel         -> 51ultron.com/competitor/
Tools/stack              -> 51ultron.com/stack/
Default                  -> 51ultron.com
```

## SUPPORT & TROUBLESHOOTING

**"Score is below 70"**
→ Redesign the visual. Weak points are listed. Usually: hook needs to be more contrarian, or visual hierarchy needs adjustment.

**"Export bar not appearing"**
→ Ensure CDN scripts loaded before </body>. Check browser console for errors.

**"HTML looks wrong"**
→ Verify inline CSS syntax. No external stylesheets allowed. Colors must be hex (#C94A22), not RGB or names.

**"Post copy doesn't feel right"**
→ Check: Is hook contrarian? Are paragraphs 10-19 words? Does it end with repost/save prompt? No banned words?
